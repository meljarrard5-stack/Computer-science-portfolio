# CS 210 Portfolio Artifact – Airgead Banking

## Project Summary

For my CS 210 portfolio artifact, I selected the Airgead Banking investment calculator project. This program was designed to help users understand how compound interest and recurring monthly deposits can increase savings over time.

The program asks the user for an initial investment amount, monthly deposit amount, annual interest rate, and number of years. It then displays yearly balances with and without additional monthly deposits.

This project solves the real-world problem of helping users visualize long-term savings growth and make better financial decisions.

---

## What I Did Particularly Well

I believe I did well organizing the project into multiple files and creating clear program structure using classes, header files, and source files.

I also did well formatting the output so the yearly reports were readable and professional. My calculations were carefully tested to ensure accurate results.

## Where I Could Enhance My Code

I could improve the program by adding stronger input validation to prevent invalid entries such as negative values or letters.

I could also improve the user experience by creating a graphical user interface instead of a console application.

Additional improvements could include exporting reports, allowing different compounding schedules, and storing customer scenarios.

These changes would improve usability, security, and efficiency.

---

## Most Challenging Pieces of Code

The most challenging part of the project was building the compound interest calculations while correctly handling monthly deposits over multiple years.

I overcame this by breaking the problem into smaller steps, testing each formula carefully, and using debugging techniques to compare expected results with actual output.

Resources I used included course materials, instructor guidance, and programming references.
